# Workspace Hygiene

ContentEngine keeps source code and runtime output separate. This matters before merging public pilot, video, and provider work into `main`.

## Source Code

Source code is safe to commit when it is intentional and reviewed:

- application code in `app/`;
- tests in `tests/`;
- scripts in `scripts/`;
- documentation in `docs/`;
- templates and static UI assets in `app/templates/` and `app/static/`;
- small sample input files that are intentionally part of the repository.

## Runtime Output

Runtime output is generated by local runs, provider calls, acceptance smoke tests, and development servers. It should stay local unless a specific small fixture is explicitly whitelisted.

Common runtime output includes:

- local SQLite databases such as `*.db`, `*.sqlite`, and `*.sqlite3`;
- generated media under `media/`;
- provider outputs and downloaded videos;
- generation reports under `media/generation_reports/`;
- local logs under `logs/` or `*.log`;
- Python caches such as `__pycache__/`, `.pytest_cache/`, and `.mypy_cache/`;
- coverage output such as `.coverage` and `htmlcov/`;
- temporary files such as `*.tmp`.

## Never Commit

Never commit:

- `.env` or `.env.*` files, except `.env.example`;
- API keys, provider secrets, OAuth tokens, signed URLs, cookies, or screenshots containing secrets;
- paid provider output videos such as `*.mp4`, `*.mov`, `*.webm`, or `*.avi`;
- local databases;
- raw media from Bombbar or other smoke tests unless a tiny fixture is explicitly whitelisted in `.gitignore` with a comment.

## Local Media And Reports

Local video and provider artifacts belong under `media/`, for example:

- `media/output/`;
- `media/provider/`;
- `media/generation_reports/`.

Keep raw videos local. When an acceptance run is useful, preserve the result as a small markdown summary or sanitized JSON excerpt in `docs/`, without provider secrets, signed URLs, or raw video files.

## Old Bombbar Smoke Outputs

Old Bombbar smoke outputs are useful as human review references, but they are not source code. Do not commit the raw video files or generated provider folders. If the learning should be preserved, write a short acceptance note in `docs/` describing:

- command used;
- provider job id if safe to expose;
- local output path;
- result summary;
- product identity issues such as packaging drift or muesli-like edible drift.

## Safe Cleanup

Preview cleanup first:

```powershell
python scripts/clean_local_artifacts.py --dry-run
```

Apply safe cleanup:

```powershell
python scripts/clean_local_artifacts.py --apply
```

Include generated media only when you are ready to remove local provider outputs and reports:

```powershell
python scripts/clean_local_artifacts.py --apply --include-media
```

The cleanup script must not delete source directories such as `app/`, `tests/`, `docs/`, `scripts/`, `sample_data/`, templates, or static source assets.
