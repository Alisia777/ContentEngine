# Контент ИИ Завод: продукт для человека без опыта

## Обещание продукта

Пользователь выбирает один товар, создаёт безопасный ролик, проверяет его, публикует с измеримой ссылкой и видит, что улучшить в следующем ролике. На каждом экране есть одно рекомендуемое действие, объяснение результата и честный статус недоступных функций.

Основной путь:

```text
Товар → Фото → Идея → Черновик без расходов → Генерация → Проверка человеком
→ Публикация → Данные площадки → Заказы и выручка → Следующая гипотеза
```

## Главный показатель

**Измеримые контент-циклы за 7 дней** — количество уникальных публикаций, для которых:

1. готовое видео одобрено человеком;
2. сохранён реальный `final_url`;
3. публикация связана с tracking-ссылкой;
4. получена хотя бы одна строка атрибутированных метрик.

Заказы и выручка показываются рядом как бизнес-результат, но не входят в определение цикла: качество товара и спрос не должны скрывать работоспособность самого контент-конвейера.

## Девять блоков

| № | Блок | Результат для новичка | Ключевая метрика |
|---|---|---|---|
| 1 | Старт и интерфейс | Понимает текущее состояние и одно следующее действие | activation rate, время до первого готового черновика |
| 2 | Генерация видео | Получает ролик по точному SKU без случайного платного запуска | provider success rate, время и стоимость генерации |
| 3 | Качество | Видит реальное видео рядом с эталоном и принимает понятное решение | first-pass approval, identity/packaging pass, regeneration rate |
| 4 | Воронка | Видит путь просмотр → переход → заказ → выручка | CTR, CVR, revenue per measurable cycle |
| 5 | Данные из сетей | Понимает источник, свежесть и достоверность каждой цифры | sync freshness, coverage, unmatched rate |
| 6 | Оплата и расходы | Разделяет расходы провайдера, выплаты участникам и оплату сервиса | cost per approved video, payout aging, gross margin |
| 7 | Артикулы Wildberries | Связывает только свои подтверждённые карточки и историю замен | matched listings, ambiguous rows, alias coverage |
| 8 | Аналитика | Понимает, какой SKU, ролик и площадка дали результат | measurable cycles, orders, revenue, winning variants |
| 9 | Обучение | Получает короткий урок непосредственно перед новым действием | completion, certification → first successful action, self-service rate |

## Активация новичка

- Первая активация за 24 часа: выбран SKU, точные фото прошли проверку, создано ТЗ/prompt-only без платного вызова.
- Ценность за 7 дней: первый ролик одобрен человеком.
- Бизнес-активация за 14 дней: первый измеримый контент-цикл.

Дополнительно считаются P50/P90 времени до каждого результата, устранение первого препятствия за 24 часа и доля пользователей, дошедших до результата без помощи оператора.

## События продукта

Отдельный append-only журнал `FactoryEvent` не смешивается с security-аудитом.

Основная воронка событий:

```text
session_started → onboarding_started → product_created/product_selected → asset_gate_passed
→ prompt_ready → generation_requested → generation_succeeded
→ human_review_completed → video_approved
→ publishing_package_approved → publication_completed
→ first_metric_attributed → first_order_attributed
```

Клиент записывает только просмотры страниц, навигацию, основное действие и безопасные UX-сигналы. Результаты провайдера, публикации, атрибуции и заказов записываются только сервером.

Защищённый `GET /api/factory-dashboard` возвращает org-scoped North Star, девять блоков, продуктовую воронку, учебный прогресс, канонические циклы, расходы и сигналы качества данных. Это стабильная точка для совместного разбора метрик с оператором или Codex; email, токены и данные другой организации в ответ не входят.

## Неподменяемые правила качества

- Placeholder, текстовый файл с расширением `.mp4` и синтетические кадры не могут открыть публикацию.
- Кадры, исходный MP4, OCR-контракт и алгоритм проверки связываются неизменяемым fingerprint-снимком; подмена файла после QA немедленно возвращает блокировку.
- Перед генерацией оператор явно переписывает 2–12 ключевых надписей упаковки. Если обязательный OCR или локальный эталон недоступен, публикация остаётся закрыта.
- Metadata score не называется качеством готового ролика.
- Итоговое решение хранит точность товара, упаковку, геометрию, сцену, CTA и личность проверяющего.
- Платный вызов требует роли, обучения, spend gate и idempotency key.
- Product UGC должен создавать канонический `VideoJob`, чтобы путь продолжался в публикацию и аналитику.

## Wildberries

«Подменный артикул» означает легальную связь собственных карточек продавца, а не скрытую подмену чужого товара.

Целевая модель:

```text
CanonicalProduct
  └─ MarketplaceListing (organization, seller account, nmID, vendorCode, barcode)
       └─ ListingAlias / ReplacementHistory (valid_from, valid_to, reason, approved_by)
```

Метрики сначала сопоставляются с listing, затем с canonical product. Неоднозначные строки попадают в quarantine. SKU/nmID обязательно входит в idempotency key.

## Оплата

Три независимых контура:

1. расходы на AI/video providers;
2. выплаты авторам и операторам;
3. клиентский billing продукта.

До появления платёжной интеграции интерфейс явно говорит «ручной расчёт». Выплата не может стать `paid` без утверждающего, времени, transaction reference и замороженной расчётной базы.

Клиентский billing уже ведёт отдельный append-only счёт, версии тарифа, инвойсы по точному `ContentCycle`, внешние платежи и связанные credit-корректировки. Эти действия не вызывают банк или эквайринг, защищены owner/admin gate и сессионным CSRF-токеном.

## Порядок реализации

### Реализовано в текущем срезе

- novice-first оболочка, девять блоков и адаптивный путь с одним следующим действием;
- append-only продуктовая телеметрия и org-scoped воронка активации;
- канонический `ContentCycle`: Product UGC → real `VideoJob` → подробный QA → manual package → tracking link → `final_url` → метрика;
- fail-closed media QA: mock, placeholder, synthetic frames и неодобренный результат не открывают публикацию;
- неизменяемые CV/OCR-снимки с source/frame hashes, freeze/duplicate проверкой, обязательными надписями упаковки и повторной верификацией при выдаче;
- durable Product UGC queue с at-most-once paid submit, supervised heartbeat, bounded retry, restart recovery и owner/admin reconciliation карантина;
- безопасный ручной/social API ingestion с cumulative replace-not-sum, idempotency и quarantine;
- официальные org-scoped адаптеры YouTube Analytics, Instagram Insights и
  TikTok Display API без хранения access token; они считаются подключёнными
  только после credential verification, Telegram/VK остаются guided manual/CSV;
- официальный org-scoped Wildberries Seller Analytics connector с HeaderApiKey
  env-reference, точным nmID mapping, семидневным периодом, pagination,
  immutable snapshots и quarantine;
- immutable ledger стоимости генерации и безопасный payout ledger без автоматических переводов;
- отдельный клиентский billing ledger: account, versioned subscription, invoice, external payment и linked credit correction без движения денег;
- MarketplaceListing/ListingAlias/ReplacementHistory для собственных карточек WB;
- семь учебных шагов и реальные мини-тесты: пустой, seeded или поддельный сертификат не открывает gate;
- invite-only Supabase membership и блокировка unscoped legacy-разделов в публичном режиме.
- org-scoped защищённая выдача MP4/кадров/asset-файлов вместо общего `/media`;
- session-bound CSRF-защита всех HTML-форм, изменяющих состояние, включая QA,
  публикацию, расходы, WB-связи, обучение, paid generation, queue reconciliation
  и клиентский billing;
- default-deny граница публичного пилота: legacy API/экраны без доказанного
  organization scope закрыты, а внешняя tracking-ссылка `/r/{slug}` остаётся
  публичной для честного подсчёта переходов.

### P0 — безопасный пилот

- единая Public Pilot оболочка и вход через `/control-room`;
- девять блоков, одно действие и понятные статусы;
- продуктовая телеметрия и базовый North Star;
- organization scoping и RBAC всех paid/review/payout/connector endpoints;
- fail-closed media QA;
- исправление уникальности метрик WB и расчёта выплат.

### P1 — сквозной завод

- мастер создания товара и пятишаговый генератор с autosave;
- следующие официальные Telegram/VK connectors поверх готового org-safe ingestion contract;
- UI разбор social/WB quarantine.

### P2 — масштабирование

- платёжный provider поверх готового billing ledger только после отдельного security/compliance этапа;
- официальные коннекторы площадок по приоритету бизнеса после YouTube;
- аналитика вариантов, стоимости и удержания;
- рекомендации следующего ролика на фактических данных.

## Критерий готовности для новичка

Новый пользователь без инструкции может за одну сессию:

1. понять, что сделать сейчас;
2. добавить или выбрать товар;
3. собрать корректные фото;
4. получить бесплатный черновик ТЗ;
5. понять стоимость до платного запуска;
6. увидеть и проверить реальный результат;
7. подготовить измеримую публикацию;
8. объяснить, откуда взялась каждая цифра в аналитике.
